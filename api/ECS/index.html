ECS

Romal's ECS module. Adds support for the ECS paradigm. Provides most common ECS features such as being able to query entities.

}

Component<T>​

</>
interface Component<T> {
Value: T
ComponentChanged: Event<T>
}

ComponentChanged is read only, Value can be written to.

ECS.CreateComponent("Test", 100)
ECS.NewEntity()

local component = ECS.GiveComponent(1, "Test")

component.ComponentChanged:Connect(function(newValue)
	print(newValue) -- Prints 300
end)

component.Value = 300

You are able to get intellisense by casting the type.

ECS.CreateComponent("Intellisense", true)
ECS.NewEntity()

local component = ECS.GiveComponent(1, "Intellisense") :: ECS.Component<boolean>

component.Value = false -- Valid
component.Value = 100 -- Type error

You will get an error if you attempt to run the following code.

ECS.CreateComponent("Errors", 0)
ECS.NewEntity()

local component = ECS.GiveComponent(1, "Errors")

component.ComponentChanged = "foo" -- Error
component.NewKey = "bar" -- Error
print(component.RealValue) -- Error

Entity<T>​

</>
interface Entity<T> {
Components: number
Instance: T?
ComponentAdded: Event<string>
ComponentRemoved: Event<string>
}

An entity table, ComponentAdded and ComponentRemoved return the name of the component that was removed when calling functions connected to them. All of these values are read only.

_ENTITY_ID_STRING​

This item is read only and cannot be modified. Read Only
</>
ECS._ENTITY_ID_STRING: string

Name of the attribute given to instances bound to an entity. The attribute holds the ID of the entity.

Query​

</>
ECS.Query(
...: string--

Names of components

) → (
number,--

ID of Entity

...any--

Value it has with the given components

)

Used in for loops. Returns the ID and the value it currently has with the given components.

ECS.CreateComponent("Test1", 100)
ECS.CreateComponent("Test2", true)
ECS.NewEntity()
ECS.NewEntity()

local id = ECS.NewEntity()

ECS.GiveComponent(id, "Test1")

for i, v in ECS.Query("Test1")
	print(i, v) -- 3 100
	-- Only the third entity is looped over
end

CreateComponent​

</>
ECS.CreateComponent(
name: string,--

Name of component

value: T--

Default value for the component

) → ()

Creates a new component with value T, the value given to it will be the default that is assigned to entities whenever they are given the component. 

ECS.CreateComponent("Test", 1000) -- 1000 is the default value
ECS.NewEntity()

local component = ECS.GiveComponent(1, "Test")
print(component.Value) -- 1000
Invalid Values

table, thread, userdata, and function will cause the function to error, these types are not allowed to be used for the value of the component.

Errors

TypeDescription
"Invalid value"Value given is an invalid value
"Exceeded maximum"There is a maximum of 32 components that can be created
